# Carbon Intelligence Platform — 開発方針プロンプト

> このドキュメントは、Carbon Intelligence Platform（https://intelligence.carboncredits.jp）の開発方針を定めたものである。事業設計書（2026年3月策定）に基づき、開発の優先順位と技術的な実装方針を定義する。
>
> 開発にあたっては、このドキュメント全体を事前に読み込んだ上で、各タスクに取り組むこと。個別の機能実装を依頼された場合も、必ずこのドキュメントの文脈（事業目的、ターゲットユーザー、提供価値の構造）を踏まえた上で判断すること。

---

## 1. 事業概要

### サービス構造
- **運営会社**: 株式会社CradleTo
- **メディアブランド**: carboncredits.jp（カーボンクレジット専門メディア）
- **有料サービス**: Carbon Intelligence（carboncredits.jpの姉妹サービス、月額5,000円）
- **URL**: https://intelligence.carboncredits.jp
- **技術スタック**: Next.js (React 19) / Vercel / WordPress (Headless CMS) / Gemini AI / Claude AI

### ファネル構造

```
carboncredits.jp（無料・広く集客）
    ↓ コンバージョン
Carbon Intelligence（月額5,000円・深い情報+ツール）
    ↓ リード顕在化
CradleToの商材群（コンサル・ツール・トレーニング・レポート等）
```

Carbon Intelligenceは「それ自体のMRR」に加えて「有料会員であること自体がリードのクオリフィケーション」として機能するミドルファネルである。月額5,000円を払っている＝カーボンクレジットに本気で取り組んでいる企業担当者、という強いシグナルになる。

## 2. ターゲットユーザー

ターゲットは「企業」ではなく「個人としてのプロフェッショナル」。月額5,000円の個人決裁ラインを前提とする。

### Persona A: 事業会社のサステナ担当者（プライマリー）
- GXリーグ参加企業（700社超）のサステナ部門 30代後半〜40代
- 「GX-ETS Phase 2対応でクレジット調達が必要になったが、何をいくらで買うべきかわからない」
- 想定母数: 700〜2,100名

### Persona B: 商社・仲介業者のクレジット担当
- 大手商社、バイウィル、Carbon EX等。クレジット売買が本業に直結
- 「市場の需給バランスと品質トレンドを即座に把握し、売買タイミングの判断に活かしたい」
- 想定母数: 40〜150名（少数だが高エンゲージメント）

### Persona C: クレジット関連スタートアップ
- CDR企業、J-Credit創出事業者、大企業のカーボンクレジット新規事業部門
- 「自社クレジットの価格設定・市場ポジショニングの根拠がほしい」
- 想定母数: 100〜200社（急成長中）

### Persona D: 金融機関・コンサルの専門家
- 銀行・保険・アセマネのESG/GX担当、コンサル・監査法人のサステナ部門
- 「クライアントへの助言に市場情報が必要だが、自社でデータを持っていない」

## 3. 提供価値の4層構造

すべての開発判断は、この4層のどこに貢献するかを基準に行う。**L2（解釈）が事業の核心**であり、最も投資すべきレイヤーである。

| レイヤー | 内容 | 削減する時間 | 重要度 |
|---|---|---|---|
| L1: 情報集約 | 散在する市場情報を一元化。価格データ、メソドロジーDB、企業DB | 「調べる時間」 | ★★★ |
| L2: 解釈 | AI+専門家解説で「だから何？」を届ける。週次ブリーフ、市場分析、政策ロードマップ | 「考える時間」 | ★★★★★ |
| L3: 意思決定支援 | 上司への説明資料としてそのまま使えるチャート・レポート。品質判断・調達タイミングの根拠 | 「稟議を通す時間」 | ★★★★ |
| L4: コミュニティ | 会員同士のネットワーキング・ウェビナー（中期以降） | — | ★（将来） |

### コアバリューの定義

Carbon Intelligenceの本質的な価値は「データの集約」ではなく「解釈付きのインテリジェンス」にある。

- ユーザーが最も知りたいのは「今何が起きているか」ではなく「だから自分は何をすべきか」
- グローバル競合（Sylvera、Xpansiv等）は「データプラットフォーム」として包括性で勝負している
- Carbon Intelligenceは「解釈プラットフォーム」としてインサイトの質で勝負する
- データの量では勝てないが、「だから何？」の深さ、日本語、日本市場の文脈で勝つ

## 4. 競合認識

### 直接競合: exroad（エクスロード）
- 日本市場における唯一のダイレクト競合
- GX-ETS・カーボンクレジット情報を日本語で提供する生成AI活用の情報サービス
- 2026年3月にアスエネグループ（Carbon EX）への参画・吸収合併を検討中と発表
- Carbon EX統合後の動向が最大のウォッチポイント

### 差別化ポイント（対exroad）
- exroadは法人向けコンサル寄り → Carbon Intelligenceは個人決裁の月額5,000円プラットフォーム
- carboncredits.jpの集客基盤（SEO・PV）はexroadが持たない独自資産
- exroadはCarbon EXに統合されることで取引所寄りのポジショニングに移行する可能性が高い

### グローバル競合（間接競合）
- Sylvera、Xpansiv/CBL、Abatable、ClearBlue Markets、Alpha Earth等
- いずれもエンタープライズ向け（年間数万ドル〜）、英語のみ、日本市場の深掘りなし
- 直接競合ではなく補完関係。同じ土俵で戦わない

## 5. 開発優先順位

### 判断基準

1. 有料会員の獲得に直結するか（ペイウォール・決済は最優先インフラ）
2. 有料会員の維持に直結するか（L2コンテンツの品質が解約率を決める）
3. 提供価値のどのレイヤーに貢献するか（L2 > L3 > L1 > L4の順）
4. ターゲットペルソナの何人に刺さるか（全ペルソナ共通 > 特定ペルソナ向け）

### 優先度S（最優先・ローンチ前に必須）

| 機能 | 目的 | 実装状況 | 対応方針 |
|---|---|---|---|
| 決済システム（Stripe） | 有料課金のインフラ | 未実装 | Stripe Checkout + Webhookで実装 |
| ペイウォール | Free/Standardの切り分け | 未実装 | 会員認証+コンテンツへのアクセス制御 |
| 週次ブリーフの品質向上 | L2キラーコンテンツ | 実装済みだが品質課題あり | AI生成→人間レビューフロー。Gemini API有料化 |
| 市場価格モニター（14市場） | L1ペイウォール必須機能 | 実装済み | データソースの信頼性向上 |
| 政策ロードマップ | L2キラーコンテンツ | 実装済み | イベント追加の運用継続 |

### 優先度A（ローンチ後1ヶ月以内）

| 機能 | 目的 | 実装状況 | 対応方針 |
|---|---|---|---|
| 特別記事・メルマガ | L2コンテンツ拡充 | 一部実装済み | 月2本ペースの制作体制 |
| AI市場分析の品質改善 | L2品質底上げ | 実装済みだが不安定 | Gemini有料化+検索API切替 |
| メソドロジー横並び比較 | L3意思決定支援 | 未実装 | 2〜3件を選んで比較するUI |
| carboncredits.jp → Intelligence導線 | ファネル転換 | 未実装 | 記事末尾にCTA設置 |
| メール配信基盤 | メルマガ・リテンション | 未実装 | SendGrid or Resend等 |

### 優先度B（ローンチ後3ヶ月以内）

| 機能 | 目的 | 実装状況 |
|---|---|---|
| メソドロジーDBの日本市場特化情報 | L1差別化 | 実装済み（200件） |
| 企業DBの強化 | L1補助 | 実装済み（198社） |
| ダッシュボードのFree/Standard表示切り分け | UX | 実装済み |
| 市場間価格チャート重ね合わせ | L3 | 未実装 |

### 優先度C（中期・3ヶ月以降）

| 機能 | 目的 |
|---|---|
| PDFレポート自動生成 | L3（Proプラン向け） |
| ポートフォリオ管理 | L3（Proプラン向け） |
| 規制コンプライアンスチェッカー | L3（SaaSツール化候補） |
| 価格アラート | L1補助 |
| ウォッチリスト | L1補助 |

### 優先度見直し

| 機能 | 理由 |
|---|---|
| ライブラリ（Google Drive連携） | PDF OCRが不安定。ニッチな付加価値。後回し |
| ダッシュボードKPIサマリー | ペイウォール前後で再設計が必要だが優先度低 |

## 6. 技術的な実装方針

### 決済・認証（最優先）

```
想定アーキテクチャ:
- Stripe Checkout Session → 決済処理
- Stripe Webhook → 会員ステータス更新
- Next.js Middleware → 認証チェック・ペイウォール制御
- WordPress側: ユーザーメタ or カスタムフィールドで会員ステータス管理
  （もしくはSupabase等の外部DBで会員管理を分離）
```

### ペイウォールの設計原則

- Free版でもダッシュボードの一部は見せる（「チラ見せ」で動機を作る）
- 市場価格は一部市場のみFreeで開放、全14市場はStandard限定
- 週次ブリーフはサマリーのみFree、全文はStandard限定
- メソドロジーDB・企業DBは一覧までFree、詳細ページはStandard限定
- 政策ロードマップ・特別記事・AI分析はStandard限定

### AI分析の品質管理

- Gemini API有料プランへの切替（ローンチ前に完了）
- DuckDuckGo→Tavily API等の有料検索APIへの切替を検討
- AI出力にステータスフラグ（draft / reviewed / published）を追加
- 生成プロンプトの改善

## 8. 開発時の判断基準

1. 「この機能は有料会員の獲得・維持にどう貢献するか？」を常に問う
2. L2（解釈）の品質が最重要
3. 「チラ見せ」の設計を意識する
4. 過剰な機能開発を避ける
5. exroadとの差別化を意識する

## 9. コンテンツのトーン・品質基準

### 週次マーケットブリーフ
- AI生成→人間レビュー→公開の3ステップ
- 「だから何？」「次に何が起きそうか？」まで踏み込む

### 特別記事
- 深掘り分析。carboncredits.jpでは出さないレベルの専門性
- 1記事あたり3,000〜5,000字程度

### AI市場分析
- 必ず人間がファクトチェック・編集してから公開
- 情報源を明示する
